# JVM 整体结构

类加载子系统加载 Class files 到内存中，生成一个大的 Class 对象

类加载子系统 -> 运行时数据区 -> 执行引擎，分为三部分的内容

执行引擎

    1. 解释执行，解释器，响应时间
    2. JIT 即时编译器，热点代码，缓存，执行的性能

指令架构

    1. 基于栈的指令集，设计实现简单，资源受限，大部分零地址指令，指令集小，无需硬件支持，移植性好，8位指令集，跨平台
    2. 基于寄存器的指令集架构，x86，性能优秀，执行高效，依赖于硬件，可移植性差，花费指令更少，一地址、二地址、三地址，16位指令集

JVM 生命周期

虚拟机启动

    引导类加载器（booststrap class loader）加载一个初始类（initial class），类（虚拟机具体实现决定）

虚拟机执行

    任务：执行 java 程序
    程序开始时虚拟机执行，程序结束虚拟机停止

虚拟机退出

    1. 正常结束
    2. 异常或者错误异常终止
    3. 操作系统错误 JVM 线程终止
    4. 程序中主动停止
